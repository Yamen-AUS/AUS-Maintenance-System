### **الخطوة 4: رفع الملفات الجديدة**

#### **4.1 - رفع `index.html`**

##### إذا كان الملف موجوداً (استبدال):
1. **انقر على `index.html`** في قائمة الملفات
2. انقر على **أيقونة القلم (Edit)** في الزاوية اليمنى
3. **احذف كل المحتوى القديم** (Ctrl+A ثم Delete)
4. **افتح الملف الذي حملته** من بيئة التطوير بمحرر نصوص
5. **انسخ كل المحتوى** (Ctrl+A ثم Ctrl+C)
6. **الصق المحتوى الجديد** في GitHub (Ctrl+V)
7. **أسفل الصفحة** اكتب رسالة commit:
   ```
   Update index.html - Complete system v3
   ```
8. اضغط **Commit changes** (الزر الأخضر)

##### إذا لم يكن الملف موجوداً (رفع جديد):
1. من الصفحة الرئيسية للمستودع، اضغط **Add file** → **Upload files**
2. **اسحب الملف `index.html`** إلى المربع المخصص
   أو اضغط **choose your files** واختر الملف
3. اكتب رسالة commit:
   ```
   Add complete index.html
   ```
4. اضغط **Commit changes**

## ❌ حل المشاكل الشائعة

### **المشكلة 1: الصفحة بيضاء فارغة**

**السبب:** الملفات لم ترفع بشكل صحيح أو الكاش لم يُمسح

**الحل:**
1. تأكد من وجود الملفات الثلاثة في المستودع
2. افتح `index.html` على GitHub وتأكد أن آخر سطرين هما:
   ```html
   <script src="app-v2.js"></script>
   <script src="pages-functions.js"></script>
   ```
3. امسح الكاش تماماً (Ctrl+Shift+Delete)
4. أعد تحميل الصفحة (Ctrl+Shift+R)
5. جرب متصفح آخر (Chrome → Firefox)
6. جرب وضع التصفح الخفي (Incognito Mode)

---

### **المشكلة 2: زر Login لا يعمل**

**السبب:** ملف `app-v2.js` غير محمّل أو قديم

**الحل:**
1. افتح Console في المتصفح (اضغط F12)
2. انقر على تبويب **Console**
3. ابحث عن أي أخطاء حمراء (Errors)
4. إذا رأيت `Failed to load resource: app-v2.js 404`:
   - الملف غير موجود في المستودع
   - ارفعه مجدداً
5. إذا رأيت `Uncaught ReferenceError`:
   - الملف قديم
   - استبدله بالنسخة الجديدة

---

### **المشكلة 3: الصفحات (Schedule, Analytics, Vendors) لا تفتح**

**السبب:** ملف `pages-functions.js` غير محمّل

**الحل:**
1. تأكد من وجود `pages-functions.js` في المستودع
2. افتح الرابط مباشرة:
   ```
   https://yamen-aus.github.io/AUS-Maintenance-System/pages-functions.js
   ```
3. إذا ظهر 404، ارفع الملف
4. إذا ظهر محتوى الملف، امسح الكاش

---

### **المشكلة 4: البيانات لا تحفظ**

**السبب:** localStorage معطّل أو الموقع مفتوح من ملف محلي

**الحل:**
1. **لا تفتح الملف محلياً** (file:/// لن يعمل localStorage)
2. استخدم الرابط على GitHub Pages فقط
3. تأكد من عدم استخدام وضع التصفح الخفي (Incognito)
4. تحقق من أن المتصفح يسمح بـ localStorage

---

### **المشكلة 5: GitHub Pages لا يتحدث**

**السبب:** التحديثات تأخذ وقتاً

**الحل:**
1. انتظر 2-5 دقائق بعد رفع الملفات
2. اذهب إلى **Settings → Pages**
3. إذا رأيت "Your site is being built" → انتظر
4. إذا رأيت "Deploy failed" → تحقق من الأخطاء
5. **حل سريع:** قم بتعديل صغير في `index.html`:
   - أضف مسافة في أي مكان
   - Commit changes
   - سيعيد بناء الموقع
